## Issue 1: PRD 07 `WriteAppliedTFVars` + Apply hook ‚Äî landed

**Severity**: medium (feature surface)
**Status**: resolved

**Context**: PRD 07 ¬ß"Goal" asks for a snapshot of the effective
var-file inputs written to the per-phase state dir after every
successful `terraform apply`. Today the var-file chain is
reconstructable only by re-running `roksbnkctl` against the same
workspace config ‚Äî losing or regenerating `config.yaml` loses the
audit trail.

**Closure**: three coordinated edits.

1. **`internal/config/applied_tfvars.go`** (new file). Public surface:
   - `WriteAppliedTFVars(workspace, phase string, sources []string) error`
   - `AppliedTFVarsPath(workspace, phase string) (string, error)` ‚Äî exposed
     so callers / tests can locate the snapshot without writing it.
   - `SetAppliedTFVarsVersion(fn func() string)` ‚Äî same import-cycle-
     dodging seam pattern as `exec.SetToolImageTag`; wired from
     `internal/cli/root.go::init` to the build-time `Version` constant
     so the header records the producing binary.

   Behavior matches PRD 07 ¬ß"Design" exactly:
   - **Path per phase**: `cluster` ‚Üí `<state-cluster>/terraform.applied.tfvars`;
     `trial` and `legacy-single` ‚Üí `<state>/terraform.applied.tfvars`.
     Unknown phases fall back to the trial path (defensive ‚Äî see PRD 07
     ¬ß"Anti-patterns to avoid" #4 spirit).
   - **Header**: `# Generated by roksbnkctl <version> at <RFC3339> after
     terraform apply on phase=<phase>.` + the "do not edit by hand" line.
   - **Section labels**: basename mapping ‚Üí `terraform.tfvars` ‚Üí
     `from config.yaml`, `terraform.tfvars.user` ‚Üí
     `from terraform.tfvars.user`, `cluster-phase-override.tfvars` ‚Üí
     `from cluster-phase override`, fallthrough ‚Üí basename.
   - **Alphabetic sort within section**.
   - **Redaction list**: exactly `ibmcloud_api_key` (PRD 07 ¬ß"Resolved
     design decisions" #4). Emitted as
     `ibmcloud_api_key = "<redacted>"  # source: cred resolver, not persisted`.
   - **Missing-source behaviour**: skipped sections get a
     `# === from <label> (missing) ===` marker + a warning to stderr.
   - **Atomic write**: temp file in same dir, chmod 0600, then rename.
     Belt-and-suspenders against half-written snapshots on signal kill.

   tfvars parsing is a minimal line-oriented regex (PRD 07's "small"
   estimate ruled out pulling in `hashicorp/hcl/v2` ‚Äî it isn't already
   in `go.mod`). Supports the line shapes roksbnkctl itself emits
   (identifier = quoted-string | bare-bool | bare-number) plus trailing
   `# ...` comment stripping for hand-edited `terraform.tfvars.user`
   files. Non-matching lines (heredocs, multi-line lists/objects) are
   silently dropped ‚Äî out of scope per PRD 07 ¬ß"What's NOT captured".

2. **`internal/tf/terraform.go::Workspace.Apply` hook**. After
   `w.tf.Apply(...)` returns nil, calls
   `config.WriteAppliedTFVars(w.name, phase, sources)` with the same
   `sources []string` it gave to terraform. Snapshot failure ‚Üí warning
   to `os.Stderr` only, returning nil up to the caller (PRD 07
   ¬ß"Anti-patterns to avoid" #4: apply succeeded, snapshot is a
   nice-to-have output).

   Phase classification lives in a new `phaseLabel(_ []string) string`
   helper:
   - `filepath.Base(w.stateDir) == "state-cluster"` ‚Üí `"cluster"`.
   - else `config.DetectShape(w.name) == ShapeLegacySingle` ‚Üí
     `"legacy-single"`.
   - else `"trial"` (the modern `bnk up` flow and the defensive
     fallback for any unclassifiable case).

   `Destroy` deliberately gets no hook ‚Äî PRD 07 ¬ß"Resolved design
   decisions" #2: a snapshot represents "what was last deployed"; the
   divergence after destroy is itself a useful signal. Added a doc
   block on the `Destroy` method spelling this out so a future reader
   doesn't "fix" the missing hook.

3. **`internal/config/applied_tfvars_test.go`** (new file). Twelve
   tests cover the PRD 07 ¬ß"Acceptance criteria":
   - `TestAppliedTFVars_PathPerPhase` ‚Äî cluster/trial/legacy-single
     routing.
   - `TestAppliedTFVars_SourceOrdering` ‚Äî three-source chain preserves
     terraform's evaluation order.
   - `TestAppliedTFVars_AlphabeticSortWithinSection` ‚Äî input `c, a, b`
     emits `a, b, c`.
   - `TestAppliedTFVars_Redaction` ‚Äî `ibmcloud_api_key` becomes
     `<redacted>` with comment; `database_password` (negative test)
     passes through verbatim.
   - `TestAppliedTFVars_Idempotent` ‚Äî fixed-timestamp render is
     byte-identical on re-run.
   - `TestAppliedTFVars_FilePermissions` ‚Äî 0600 (skipped on filesystems
     that don't honor POSIX mode bits ‚Äî wsl2 `/mnt/c/` snaps to 0777;
     real Linux fs catches regressions).
   - `TestAppliedTFVars_MissingSourceFile` ‚Äî missing source becomes
     `# === from <name> (missing) ===` section.
   - `TestAppliedTFVars_AllFourShapesCover` ‚Äî Empty / ClusterOnly /
     Split / LegacySingle each routed correctly; Empty produces neither
     file.
   - `TestAppliedTFVars_DestroyDoesNotMutate` ‚Äî write once, "destroy"
     (a no-op on the snapshot ‚Äî Destroy has no hook), re-read; bytes +
     mtime unchanged.
   - `TestAppliedTFVars_HeaderRecordsPhase` ‚Äî header `phase=‚Ä¶` token
     matches the input phase string for all three labels.
   - `TestAppliedTFVars_SkipsHCLComments` ‚Äî `#`/`//`/blank lines in a
     source tfvars are skipped.
   - `TestAppliedTFVars_StripsTrailingComment` ‚Äî `region = "ca-tor" # note`
     emits `region = "ca-tor"` cleanly.

## Issue 2: minimal HCL parser scope ‚Äî non-matching lines dropped silently

**Severity**: low
**Status**: wontfix

**Context**: the tfvars line scanner in
`readTFVarsAssignments` (`internal/config/applied_tfvars.go`) is a
minimal regex matching `identifier = value` where `value` is a quoted
string, bare bool, or bare number. Any line that doesn't match ‚Äî HCL
heredocs (`<<EOF`), multi-line lists (`[ ‚Ä¶ `), object literals (`{ ‚Ä¶ }`)
‚Äî is silently dropped from the snapshot.

**Why this is OK for v1.4.0**: PRD 07 ¬ß"What's NOT captured" rules out
the broader "effective variable values terraform resolved" surface;
the snapshot is "what var-files said," and roksbnkctl's own tfvars
emitter (`internal/tf/vars.go`) only ever writes the supported line
shapes. `terraform.tfvars.user` is documented (chapter 6) as
line-oriented; the architect's chapter-6 ¬ß"Sprint 11" subsection can
add a note that exotic HCL shapes won't survive the round-trip if
that's a meaningful concern for users.

**Action**: none for v1.4.0. If validator's live verify exposes a real
user-tfvars file with a multi-line shape that's getting silently
dropped, Sprint 12 can either widen the parser (pull `hashicorp/hcl/v2`
into go.mod) or emit a `# === from <label> (skipped: unsupported HCL
shape) ===` marker. Currently this is a YAGNI-bounded concern.

### Closure

Flipped from `open (acceptable for v1.4.0)` to `wontfix` at sprint
close ‚Äî the rationale above is unchanged. PRD 07 ¬ß"What's NOT
captured" rules the broader "effective variable values terraform
resolved" surface out; roksbnkctl's own tfvars emitter
(`internal/tf/vars.go`) only ever writes the line-oriented assignment
shapes the minimal regex already handles; and the YAGNI bound holds
absent a concrete user-tfvars file that exercises a multi-line shape.
Sprint 12 trigger: if validator's live verify (or a user bug report)
surfaces a real `terraform.tfvars.user` with a heredoc / multi-line
list / object literal that's getting silently dropped, re-open and
widen the parser or emit the `(skipped: unsupported HCL shape)`
marker.

## Issue 4: snapshot does NOT participate in the next apply's var-file chain

**Severity**: low (anti-pattern guard)
**Status**: resolved (matches PRD 07 ¬ß"Anti-patterns to avoid" #1)

**Context**: the snapshot lives next to the per-phase state files
(`<state[-cluster]>/terraform.applied.tfvars`) and shares the
`*.tfvars` extension. A casual reader could assume terraform's
auto-pickup of `*.auto.tfvars` applies here too ‚Äî it does not (the
filename is `terraform.applied.tfvars`, not `*.auto.tfvars`, so
terraform ignores it). PRD 07 ¬ß"Anti-patterns to avoid" #1 forbids
adding the snapshot to `Workspace.varFiles`. Verified by inspection:
`varFiles` returns `[TFVarsPath, UserTFVarsPath?, extras...]`; nothing
in this Sprint 11 patch touches that chain.

**Action**: none. Regression guard: a future change that adds the
snapshot to the var-file chain would re-write `ibmcloud_api_key`'s
value as the literal string `<redacted>`, which terraform would then
pass to the IBM provider ‚Äî easy to spot in any e2e run.
