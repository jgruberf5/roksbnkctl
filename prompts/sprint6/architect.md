### Chapter 26 — `book/src/26-troubleshooting.md` — "Troubleshooting"

Common failure modes from real deployments. Each entry: **symptom → root cause → fix**. Mine for real failures from:
- `issues/` directory (Sprint 0-5 issue logs and resolved logs)
- `docs/SHAKEOUT.md` (real-cluster shake-out items)
- chapter 26's existing stub (likely empty)
- inferred from the e2e plan's "risks" sections per phase

Entries to include at minimum:

- **terraform-exec retries on transient IBM Cloud API errors** — `Error: timeout while waiting for state to become...`; root cause: IBM Cloud control-plane occasional 5-min API lag; fix: re-run `roksbnkctl up` (terraform state is durable; the retry is safe).
- **ROKS cluster propagation lag** — `roksbnkctl up` returns success but `roksbnkctl k get nodes` says `0 nodes`; root cause: cluster created in IBM Cloud but worker provisioning takes 5-10 min; fix: `roksbnkctl wait cluster --ready` (if it exists; else just wait).
- **Kubeconfig fetch 404** — `roksbnkctl up` post-apply hook fails fetching admin kubeconfig; root cause: ROKS cluster reports `provisioning` for ~30s after the create API returns; fix: the binary already retries with backoff; if it still fails, `roksbnkctl init --refresh-kubeconfig`.
- **OpenShift SCC violation on test pods** — `iperf3 pod fails to start: violates PodSecurity restricted-v2`; root cause: the throughput suite uses `networkstatic/iperf3` by default which runs as root; fix: workspace `test.throughput.image: ghcr.io/jgruberf5/roksbnkctl-tools-iperf3:v0.9.0` (chapter 22 covers this). Cross-link to chapter 22.
- **Workspace-delete current-workspace gotcha** — `roksbnkctl ws delete <name>` of the currently-active workspace leaves stale env var; root cause: workspace-context isn't refreshed automatically; fix: `roksbnkctl ws use <other>` before deleting.
- **Docker daemon unreachable** — exit 127 on `--backend docker`; fix: start dockerd; for rootless docker, `systemctl --user start docker`.
- **K8s backend "ops pod not found"** — exit 127 on `--backend k8s`; fix: `roksbnkctl ops install` first.
- **SSH backend "tool not found, run with --bootstrap"** — exit 127; fix: `roksbnkctl --backend ssh:<target> --bootstrap <command>` for the first call; subsequent calls skip the install check.
- **`roksbnkctl test dns` returns NXDOMAIN against an internal GSLB record** — root cause: laptop's resolver doesn't have a route to the internal GSLB VIP; fix: `--server <gslb-vip-ip>` explicitly, or use `--backend k8s` to query from inside the cluster.

Aim for 15-25 entries; quality over quantity. Forward-link to chapter 23 for the e2e plan; cross-link entries to relevant deep-dive chapters (14 for cred issues, 17 for backend mechanics, 19 for ops pod, 21 for DNS, 22 for throughput).
