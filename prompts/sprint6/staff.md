### Priority 1 — Auto-generators for chapters 27 + 29

#### 1a. `tools/refgen/cobra-md/main.go` — cobra → markdown

A small Go program that walks the `roksbnkctl` cobra command tree starting at the root command, emitting one markdown section per command. Shape:

```go
package main

import (
    "github.com/spf13/cobra"
    // ... import the root command builder from cmd/roksbnkctl
)

func main() {
    root := buildRootCommand()
    fmt.Println("# Command reference")
    fmt.Println()
    fmt.Println("Auto-generated by `go run ./tools/refgen/cobra-md > book/src/27-command-reference.md`. Re-run on every CLI surface change.")
    fmt.Println()
    renderCommand(root, /* path */ "roksbnkctl", /* depth */ 2)
}

func renderCommand(c *cobra.Command, path string, depth int) {
    // ## `<path>` — short description
    // synopsis (Use)
    // long description (Long)
    // flags table (Flags + InheritedFlags)
    // examples (Example)
    // subcommands (recurse with depth+1)
}
```

The output should be:

- One H2 (`##`) per top-level command (`roksbnkctl ws`, `roksbnkctl init`, `roksbnkctl up`, etc.); one H3 (`###`) per subcommand; H4 per sub-subcommand.
- A flag table per command: `| Flag | Default | Description |`. Skip global/inherited flags repeated at every level (render them once at the top of the chapter).
- A back-link to the parent command (`← back to [\`roksbnkctl ws\`](#roksbnkctl-ws)`).
- A short "Examples" code block, parsed from the cobra command's `Example:` field, if present.

The architect runs `go run ./tools/refgen/cobra-md > book/src/27-command-reference.md` at integration time and commits the output.

### Priority 4 — Phase N Go-side stubs (mixed-mode lifecycle)

Validator's `scripts/e2e-test-backends.sh` Phase N exercises a mixed-mode lifecycle: `up` runs with one backend, `test throughput` with another, `down` with a third. Each transition must preserve workspace state (kubeconfig, terraform state, secrets) correctly.

Go-side prerequisites:

- **Backend-independent state path** — verify that `~/.roksbnkctl/<workspace>/state/terraform.tfstate` lands at the same path regardless of which backend wrote it. Sprint 5 staff already ensured this for `--backend docker` (bind-mount at `/state`); confirm `--backend local` and any future backends agree.
- **Backend-independent kubeconfig location** — `~/.kube/config` or `~/.roksbnkctl/<ws>/state/kubeconfig` (chapter 14 §"kubeconfig discovery"). Both backends should write/read from the same paths.
- **Cred-resolver invariance** — `cred.Resolver.IBMCloudAPIKey(ctx)` returns the same value regardless of which backend invokes it; the resolver is local-to-the-host always (the cred is propagated to the backend, not resolved by it). Verify via a focused unit test (`internal/cred/resolver_invariance_test.go`).
- **Backend-switch idempotency** — running the same operation twice with different backends (`up --backend local; up --backend docker`) should be a no-op the second time, not a state corruption. The terraform-exec local backend and the terraform-docker bind-mount path both write to the same `.tfstate`; this is correct (verified by Sprint 5 staff) but file an issue if a regression test reveals a corner case.

The Go-side scope for Sprint 6 is **verification + small fix-ups** — not new code. If a backend-switch test reveals a bug, file the issue and let the validator surface it in Phase N's e2e steps.

Sprint 5 tech-writer Issue 4 + validator Issue 4 noted that PRD 03 specs the `edns_client_subnet` field but the implementation doesn't emit it. Adding it is small (~20 LOC):

```go
// In internal/test/dns.go
type DNSProbeResult struct {
    // ... existing fields ...
    EDNSClientSubnet *EDNSClientSubnet `json:"edns_client_subnet,omitempty"`
}

type EDNSClientSubnet struct {
    Family       int    `json:"family"`        // 1 = IPv4, 2 = IPv6
    SourceNetmask uint8 `json:"source_netmask"`
    ScopeNetmask  uint8 `json:"scope_netmask"`
    Address       string `json:"address"`
}

// In Probe.Run, after Exchange returns:
if opt := resp.IsEdns0(); opt != nil {
    for _, e := range opt.Option {
        if ecs, ok := e.(*dns.EDNS0_SUBNET); ok {
            result.EDNSClientSubnet = &EDNSClientSubnet{
                Family:        int(ecs.Family),
                SourceNetmask: ecs.SourceNetmask,
                ScopeNetmask:  ecs.SourceScope,
                Address:       ecs.Address.String(),
            }
            break
        }
    }
}
```
